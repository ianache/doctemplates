<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser user -> React state | User-selected page-size value crosses from a native select event into client-side pagination state. |
| React state -> local slice/rendering | Page and pageSize determine which already-fetched rows are displayed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01-T | Tampering | `Pagination` select event | mitigate | Only render fixed numeric options supplied by current page code and convert with `Number(event.target.value)` before invoking the numeric callback. |
| T-15-02-D | Denial of Service | Client-side rendering | accept | Options are limited to `[5, 10, 20, 50]`; no unbounded user input or server query amplification is introduced. |
| T-15-03-I | Information Disclosure | PagedTable consumers | accept | The phase changes only client-side display count for rows already fetched and visible under existing permissions. |
| T-15-04-E | Elevation of Privilege | Frontend pagination controls | accept | No auth, role, route, BFF, or backend access-control path is changed. |
| T-15-SC | Tampering | npm package supply chain | accept | No new packages or package-manager installs are required for this phase. |
</threat_model>
