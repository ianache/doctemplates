<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser UI -> authenticated API | User-supplied filters and button actions trigger API calls. |
| Clipboard -> outside platform | The signed public URL leaves the authenticated session boundary. |
| PDF iframe -> browser renderer | Authenticated PDF content is embedded in the app shell. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-08 | Information Disclosure | share-copy UI | mitigate | Copy only the backend-generated signed URL; do not construct signatures in the browser. |
| T-09-09 | Tampering | filter form | mitigate | Treat filters as query params only and rely on backend validation; omit blank values. |
| T-09-10 | Repudiation | UI download/share actions | transfer | Backend plan 09-02 logs authoritative download/share events server-side. |
| T-09-11 | Information Disclosure | PDF iframe | accept | Detail page is inside authenticated shell; public access is only via signed URL from backend. |
| T-09-SC | Tampering | package installs | accept | No package-manager installs are planned. |
</threat_model>
