<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> authenticated `/api/issuances` | Authenticated users provide filter values and trigger download/share audit events. |
| Internet -> `/api/public/document-issuances/{id}/download` | Anonymous callers can access files only with a valid HMAC signature. |
| Filesystem -> API response | Stored issuance paths are served as PDFs and must not be path-controlled by request input. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-03 | Spoofing | public download endpoint | mitigate | Verify HMAC-SHA256 signature with `hmac.compare_digest` before file lookup/response. |
| T-09-04 | Tampering | search filters | mitigate | Use SQLAlchemy query parameters and UUID/date parsing; never build SQL strings from query text. |
| T-09-05 | Repudiation | download/share endpoints | mitigate | Insert server-side `download` and `share` tracelog rows with authenticated or anonymous user context and request metadata. |
| T-09-06 | Information Disclosure | public URL generation | mitigate | Public URL includes opaque UUID plus HMAC signature; bad signatures return 403 and do not reveal file content. |
| T-09-07 | Denial of Service | broad issuance list | accept | MVP dataset is local/small; order newest first. Add pagination only if later requirements introduce scale constraints. |
| T-09-SC | Tampering | package installs | accept | No package-manager installs are planned. |
</threat_model>
