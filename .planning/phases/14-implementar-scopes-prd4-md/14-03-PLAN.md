<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Redis broker -> worker | Operational queue messages trigger backend code in the worker process. |
| Worker -> Postgres | Worker reads persisted issuance data and writes lifecycle changes. |
| Worker -> StorageProvider | Worker writes final PDFs to local or S3-compatible storage. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-03-01 | Tampering | Celery task arguments | mitigate | Task accepts only `issuance_id`; all payload and metadata are read from Postgres. |
| T-14-03-02 | Repudiation | worker lifecycle changes | mitigate | Persist `queued_at`, `started_at`, `completed_at`, status, task id, and success tracelog. |
| T-14-03-03 | Information Disclosure | worker errors/logging | mitigate | Truncate `error_message` and do not log full `input_data` or metadata payloads. |
| T-14-03-04 | Denial of Service | duplicate Redis redelivery | mitigate | Idempotency guard exits unless DB status is exactly `queued`; prefetch is limited to one. |
| T-14-03-SC | Tampering | `celery[redis]` runtime dependency | mitigate | Depends on plan 14-01 package verification and plan 14-02 dependency installation. |
</threat_model>
