<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> BFF | User actions request generation status, downloads, and share URLs through authenticated session proxy. |
| BFF -> Backend | BFF forwards authenticated requests and backend status/error responses. |
| Backend -> Browser | Failure messages and job state are rendered in UI. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-04-01 | Spoofing | BFF async proxy responses | mitigate | Preserve backend status codes and authenticated proxy behavior; no client-side state can create success. |
| T-14-04-02 | Information Disclosure | failed job error display | mitigate | Display bounded backend `error_message`; backend truncation is required by worker plan. |
| T-14-04-03 | Tampering | disabled actions in stale UI | mitigate | UI disables actions before success and backend 409 remains authoritative if stale UI attempts an action. |
| T-14-04-04 | Denial of Service | polling | mitigate | Poll every 2 seconds for 60 seconds, then every 5 seconds, and stop at terminal states. |
| T-14-04-SC | Tampering | npm package installs | accept | No new frontend package installs are planned. |
</threat_model>
