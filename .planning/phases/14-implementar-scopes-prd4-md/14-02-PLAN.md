<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client/BFF -> Backend API | Authenticated user input crosses into generation enqueue and issuance status endpoints. |
| Backend API -> Redis broker | Backend sends a task identifier into operational queue infrastructure. |
| Backend API -> Storage | Download/share/preview endpoints read stored PDFs only when issuance state is ready. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-02-01 | Tampering | generation enqueue payload | mitigate | Backend validates auth, design, draft activation, metadata, and persisted payload before enqueue; worker receives only `issuance_id`. |
| T-14-02-02 | Information Disclosure | share/public download before PDF readiness | mitigate | Gate share and public download on `status == "success"` and non-null `storage_key`. |
| T-14-02-03 | Denial of Service | synchronous PDF render in request | mitigate | Remove request-path `generate_composed_pdf` call and return 202 after enqueue. |
| T-14-02-SC | Tampering | `celery[redis]` package install | mitigate | Depends on blocking human package verification in plan 14-01. |
</threat_model>
