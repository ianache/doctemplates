<threat_model>
### Trust Boundaries Map
1. Untrusted Client <---[HTTP Requests with Auth JWT/Cookie]---> FastAPI backend (Routers)
2. FastAPI backend <---[SQLAlchemy ORM / SQL]---> Postgres Database (issuances, user logs)
3. FastAPI backend <---[Python standard library Path/write]---> Local Storage (.content-storage/issuances)

### STRIDE Threat Register
| Threat ID | Category | Description | Mitigation Plan |
|---|---|---|---|
| T-01 | Spoofing | Client issues documents under another user's identity or access unauthorized designs. | Gated by `get_current_user` dependency in FastAPI router. Active user ID (`user.id`) is stored in `DocumentIssuance.user_id`. |
| T-02 | Tampering | User tries to access files outside the storage root using path traversal attacks. | Strict UUID validation on issuance IDs. Use Python `Path` relative to the configured storage root to avoid path traversal. |
| T-03 | Repudiation | Operational user claims they didn't generate a document, or backend fails to audit generated legal PDFs. | Every generation logs the input payload (`input_data`), active user (`user_id`), creation timestamp, and version in `DocumentIssuance`. |
| T-04 | Info Disclosure | Unauthorized user downloads generated PDF issuances of other users. | Download API `/api/issuances/{issuance_id}/download` is authenticated and restricted. |
| T-05 | DoS | Malicious payloads or infinite loops in templates exhaust system resources. | Payload is strictly schema-validated and coerced to prevent malformed data. Jinja2 templates are strictly validated at activation time. Standard typography is restricted to Helvetica, Times, Courier. |
| T-06 | Privilege Elevation | Jinja template injection executing arbitrary Python commands. | Use Jinja2 environment in a sandboxed mode or simple parsing (standard Environment sandbox/no-extensions) to ensure only predefined date formatting filters and variables are executed. |
</threat_model>
