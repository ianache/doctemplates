# Plan: Phase 11 - BFF Component & Core Backend Isolation

## Objectives
Introduce a Backend For Frontend (BFF) service to isolate the Core Backend from browser session management. Move the OIDC authentication callbacks, login redirects, and session cookie handling to the BFF, and secure the Core Backend to only validate OIDC Bearer tokens.

<task type="auto">
  <name>Reverse Proxy & Silent Refresh Middleware</name>
  <files>
    <file>bff/app/proxy/router.py</file>
  </files>
  <read_first>
    <file>bff/app/auth/session.py</file>
    <file>bff/app/config.py</file>
  </read_first>
  <action>
    Create `bff/app/proxy/router.py` containing a catch-all endpoint `/{path:path}` that proxies requests to the Core Backend:
    - Intercepts requests. Reads and decrypts the `bff_session` cookie if present.
    - If access token is expired, performs a backchannel request to Keycloak `/token` using the refresh token to obtain a new access & refresh token. Encrypts and updates the cookie. Use an async serialization mechanism (specifically storing `asyncio.Lock` instances in a `weakref.WeakValueDictionary` scoped per session/user) to serialize token refresh operations, avoiding duplicate refresh calls and preventing memory leaks.
    - If refresh fails, deletes the `bff_session` cookie and returns HTTP `401 Unauthorized` with a details payload.
    - If a valid `bff_session` cookie is missing or invalid, check if the incoming request contains an `Authorization` header. If it does, forward the `Authorization` header unchanged to the Core Backend. If no session cookie exists and no `Authorization` header is present, proxy the request without appending an `Authorization` header.
    - If session is valid (or successfully refreshed), injects `Authorization: Bearer <access_token>` into the headers.
    - Strips all CORS-related headers (`Access-Control-*` headers) returned by the Core Backend to prevent duplicate CORS headers from blocking browser requests.
    - Strips the `host` and `cookie` headers, then streams request/response payloads using the shared `httpx.AsyncClient` from `request.app.state.client` with `content=request.stream()` and returns a `StreamingResponse`. Do NOT instantiate `httpx.AsyncClient` per request.
  </action>
  <verify>
    <automated>python -c "from bff.app.proxy.router import router; print(router)"</automated>
  </verify>
  <done>
