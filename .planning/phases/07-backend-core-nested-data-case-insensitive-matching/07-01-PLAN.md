    - API POST `/api/document-types` returns `422 Unprocessable Entity` or `400 Bad Request` when trying to define a schema with structural conflicts (e.g., `cliente` and `cliente.nombre`, or `cliente.contactos[]` and `cliente.contactos.nombre`).

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-01 | Denial of Service | Schema validation / Parser | Medium | mitigate | Limit maximum path depth in schemas to 5 levels (e.g. `a.b.c.d.e`) and enforce max index or recursion limits. |
| T-07-02 | Elevation of Privilege | Case-insensitive proxy dictionary | High | mitigate | Prevent double-underscore private attribute lookup (`__class__`, etc.) by raising `AttributeError` explicitly in wrappers. |
| T-07-03 | Tampering | Payload uniqueness checks | High | mitigate | Structured collision rejection at API entry (D-03/D-04). |
</threat_model>
