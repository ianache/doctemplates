- **Verification:** `git diff --stat pyproject.toml` correctly errors "not in the working tree", confirming no accidental modification

**2. [Rule 3 - Blocking] `uv add` failed with TLS certificate error**
- **Found during:** Task 1
- **Issue:** `uv add ...` failed with `invalid peer certificate: UnknownIssuer` when fetching from PyPI (corporate/local TLS interception)
- **Fix:** Re-ran with `--native-tls` flag on all `uv add`/`uv run` invocations
- **Files modified:** none (environment flag, not a file change)
- **Verification:** Packages installed successfully with `--native-tls`
- **Committed in:** N/A (tooling flag, not a code change)

**3. [Rule 3 - Blocking] Autogenerated Alembic migration requires a live DB connection**
- **Found during:** Task 2
- **Issue:** `alembic revision --autogenerate` hung indefinitely attempting to connect to Postgres at `localhost:5432`, which isn't running until 01-02-PLAN's docker-compose is up
- **Fix:** Hand-wrote `backend/alembic/versions/0001_initial_users_sessions.py` with explicit `op.create_table` calls mirroring the `User`/`Session` models exactly, instead of running the autogenerate command; verified via `alembic heads` (no DB connection required) that the migration is correctly registered
- **Files modified:** `backend/alembic/versions/0001_initial_users_sessions.py`
- **Verification:** `alembic heads` prints `0001 (head)`; `Base.metadata.tables.keys()` matches the migration's tables
- **Committed in:** `9d6e943` (Task 2 commit)

**4. [Rule 3 - Blocking] Stray `VIRTUAL_ENV` environment variable broke `uv run pytest`**
- **Found during:** Task 3
- **Issue:** The shell environment had `VIRTUAL_ENV` pointing to a path from a different worktree/repo location (`D:\...\29-DocManagemet\.venv`, which doesn't exist in this worktree). This caused `uv run pytest ...` to fail with `error: Failed to spawn: pytest / Access is denied.`, while `uv run python -m pytest` and direct `.venv/Scripts/pytest.exe` both worked
- **Fix:** `unset VIRTUAL_ENV` before invoking `uv run pytest` - purely a shell-session environment fix, no code/config change
- **Files modified:** none
